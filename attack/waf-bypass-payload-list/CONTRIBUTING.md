# Contributing to WAF Bypass Payload List

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to `waf-bypass-payload-list`. These are mostly guidelines, not rules. Use your best judgment and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs / False Positives

This section guides you through submitting a bug report.

*   **Use a clear and descriptive title** for the issue to identify the problem.
*   **Describe the exact steps to reproduce the problem** in as many details as possible.
*   **Provide specific examples** to demonstrate the steps.

### Suggesting Enhancements / New Payloads

This section guides you through submitting an enhancement suggestion for WAF Bypass Payload List, including completely new features and minor improvements to existing functionality.

*   **Use a clear and descriptive title** for the issue to identify the suggestion.
*   **Provide a step-by-step description of the suggested enhancement** in as many details as possible.
*   **Explain why this enhancement would be useful**.

### Pull Requests

1.  Fork the repo and create your branch from `main`.
2.  If you've added code/payloads that should be tested, add tests.
3.  Ensure your code/text formats consistent with the rest of the project.
4.  Issue that pull request!

## Style Guide

*   **Payload Format:** Please ensure payloads are one per line in text files.
*   **Comments:** If a payload requires specific context (e.g., "Only works on Nginx"), please add a comment in the file using standard comment syntax if the file type supports it, or document it in the PR.
