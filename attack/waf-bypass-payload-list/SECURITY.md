# Security Policy

## Reporting a Vulnerability

We take the security of this project and its users seriously. If you have found a vulnerability or a potential security issue in this repository or the payloads provided, please do **not** open a public issue.

### Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |

### Reporting Process

Please verify the vulnerability and report it to the maintainer directly.

1.  **Email:** Contact the project maintainer at `ismailtasdelen@protonmail.com` (or the email listed on the GitHub profile).
2.  **Details:** Please include comprehensive details about the vulnerability, including:
    *   Description of the issue.
    *   Steps to reproduce.
    *   Potential impact.
    *   Any Proof of Concept (PoC) code or payloads.

We will acknowledge receipt of your report within 48 hours and will strive to provide a fix or mitigation as soon as possible.

### Disclosure Policy

*   Please do not disclose the vulnerability publicly until it has been addressed.
*   We aim to be transparent and will credit researchers who responsibly disclose vulnerabilities.

## Disclaimer

This repository contains security payloads intended for authorized testing only. The maintainers are not responsible for the misuse of the information provided herein.
